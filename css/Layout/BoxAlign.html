# 盒子对齐

# 主轴

主轴（行内轴）

flex-direction 属性参数 方向 开始位置 结束位置
row 水平（x 轴） 左 右
row-reverse 水平（x 轴） 右 左
column 垂直（Y 轴） 上 下
column-reverse 垂直（Y 轴） 下 上

# 容器对齐排列

justify-content : normal | <content-distribution> | <overflow-position>? [ <content-position> | left | right ]
where
/* 分布位置 */
<content-distribution> = space-between | space-around | space-evenly | stretch
/* 溢出位置 */
<overflow-position> = unsafe | safe
/* 内容位置 */
<content-position> = center | start | end | flex-start | flex-end

/* Overflow alignment */
justify-content: safe center;
justify-content: unsafe center;
属性值 描述
<content-position> 位置对齐
start | flex-start 从主轴开始位置排列对齐
end | flex-end 从主轴结束位置排列对齐
center 从主轴中间位置排列对齐
left 左对齐
right 右对齐
<content-distribution> 分布式对齐
space-between 俩端各分配盒子，剩余盒子平均分配剩余空间
space-around 每个盒子分配相同的空间，但盒子之间的间隔比盒子与边框的间隔大一倍。
space-evenly 盒子之间距离平均分配
stretch 盒子会平均拉伸或伸缩 填充整个容器
<overflow-position> 溢出对齐
safe 与对齐关键字一起使用，如果选定的关键字会导致元素溢出容器造成数据丢失，那么将会使用 start 代替它
unsafe 与对齐关键字一起使用，无论盒子和对齐容器的相对大小如何，并且无论是否会发生导致数据丢失的溢出，都应遵守给定的对齐值
<Baseline-position> 基线对齐
first baseline 第一个盒子与 start 位置基线对齐
last baseline 最后一个盒子与 end 位置基线对齐

# 多个元素对齐

所有盒中的项目定义了默认的 justify-self ， 可以使这些项目以默认方式沿适当轴线对齐到每个盒子

布局 对齐方式
块级布局 将内联轴（水平）上的元素在其包含的块内对齐
绝对定位 将内联轴上（水平）的包含块内的元素对齐齐，并考虑上，左，下和右的偏移值
表单单元格 忽略
弹性布局 忽略
网格布局 行内的元素在其网格区域内对齐
justify-items ： normal | stretch | <baseline-position> | <overflow-position>? [ <self-position> | left | right ] | legacy | legacy && [ left | right | center ]
where
<baseline-position> = [ first | last ]? baseline
<overflow-position> = unsafe | safe
<self-position> = center | start | end | self-start | self-end | flex-start | flex-end
属性值 描述
auto 使用的值是justify-items父级框的属性值，除非该框没有父级，或在这些情况下位于绝对位置auto表示normal。
normal 默认为 start 属性进行位置排列
<self-position> 自我对齐
start | flex-start|self-start 元素在主轴线上，朝着对齐容器开始位置进行对齐
end | flex-end|self-end 元素在主轴线上，朝着对齐容器结束位置进行对齐
center 元素在主轴线上，朝着对齐容器中间位置进行对齐
left 元素在对齐容器左对齐
right 元素在对齐容器右对齐
stretch 元素在对齐容器里自适应填充
<overflow-position> 溢出对齐
safe 与对齐关键字一起使用，如果选定的关键字会导致元素溢出容器造成数据丢失，那么将会使用 start 代替它
unsafe 与对齐关键字一起使用，无论盒子和对齐容器的相对大小如何，并且无论是否会发生导致数据丢失的溢出，都应遵守给定的对齐值
<Baseline-position> 基线对齐
first baseline 第一个盒子与 start 位置基线对齐
last baseline 最后一个盒子与 end 位置基线对齐

<overflow-position> 溢出对齐
safe 与对齐关键字一起使用，如果选定的关键字会导致元素溢出容器造成数据丢失，那么将会使用 start 代替它
unsafe 与对齐关键字一起使用，无论盒子和对齐容器的相对大小如何，并且无论是否会发生导致数据丢失的溢出，都应遵守给定的对齐值
<Baseline-position> 基线对齐
first baseline 第一个盒子与 start 位置基线对齐
last baseline 最后一个盒子与 end 位置基线对齐

# 元素之间的间距

# 行间距

row-gap：normal | <length-percentage>
where
<length-percentage> = <length> | <percentage>
属性值 描述
normal 默认为 0 宽度间距
<length> 长度值数据类型 间距
<percentage> 百分比数据类型 间距

# 列间距

column-gap：normal | <length-percentage>
where
<length-percentage> = <length> | <percentage>
属性值 描述
normal 默认为 0 宽度间距
<length> 长度值数据类型 间距
<percentage> 百分比数据类型 间距

# 简写属性

gap： <'row-gap'> <'column-gap'>?

/* 长度值数据类型 */
gap: 20px;
gap: 1em;
gap: 3vmin;
gap: 0.5cm;

/* 百分比数据类型 */
gap: 16%;
gap: 100%;

/* 两个长度值数据类型 */
gap: 20px 10px;
gap: 1em 0.5em;
gap: 3vmin 2vmax;
gap: 0.5cm 2mm;

/* 两个百分比数据类型 */
gap: 16% 100%;
gap: 21px 82%;

/* 函数值 */
gap: calc(10% + 20px);
gap: calc(20px + 10%) calc(10% - 5px);
属性值 描述
normal 默认为 0 宽度间距
<length> 长度值数据类型 间距
<percentage> 百分比数据类型 间距
